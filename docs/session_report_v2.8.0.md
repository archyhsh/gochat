# GoChat v2.8.0 开发进度汇报 (2024-03-21)

今日完成了从 **v2.3.0 到 v2.8.0** 的深度架构重构与功能闭环，核心目标是提升系统的**高并发扩展性**、**数据一致性自愈能力**及 **Premium 级交互体验**。

## 1. 架构演进：定向路由推送 (Targeted Push)
- **重构内容**：彻底废弃了低效的“网关广播模式”，移除了网关对 Kafka 的盲目消费。
- **现行逻辑**：`Message-RPC` 作为唯一消费中枢，通过 `pkg/router` (Redis) 精准寻址，反向调用网关的 `internal/push` 接口进行定向分发。
- **价值**：极大降低了多节点环境下的 CPU 与带宽冗余，支持网关层无限水平扩展。

## 2. 一致性方案：多级版本号自愈 (Versioning)
- **设计决策**：引入基于 `UnixNano`（纳秒戳）的五级版本控制体系（User, Group, Member, Relation, Conversation）。
- **同步机制**：采用了 **“数据搭便车 (Piggybacking)”** 策略。资料变更不再触发全网广播，而是随聊天消息下发版本指纹。
- **前端自愈**：`app.js` 具备了“见信即对账”能力。发现版本落后时，静默触发局部拉取，确保了高并发场景下数据最终一致性。

## 3. 功能闭环与社交优化
- **智能三位一体搜索**：搜索栏根据 `Chats/Friends/Groups` 视图自动分流搜索逻辑，支持“搜出并复活 (Restore)”已隐藏会话。
- **高级邀请系统**：废弃了 ID 手填，重构为“好友勾选”模态框。实现了对“自己”及“已在群成员”的智能过滤。
- **实时信令系统**：实现了 Type 10-15 号信令，涵盖好友申请、拒绝、踢人、解散及多端备注同步。
- **群资料精修**：实现了群内昵称独立设置，并支持 `群昵称 (原昵称)` 的智能渲染。

## 4. UI/UX 视觉重塑
- **布局修复**：彻底解决了侧边栏“上下堆叠”问题，恢复了专业的水平名片布局。
- **细节对齐**：通过 CSS 相对定位锁定了红点位置，利用 Flex 约束消除了成员列表的重叠挤压。
- **乐观更新**：实现了消息发送的“瞬时上屏”状态（Sending...），极大提升了交互手感。
